Docker 你好 請問你是?

Posted on 2022-02-22 In Docker

Docker 是一個可以讓你的應用跑在虛擬系統上的應用程式

像是當你需要在程式裡與MySQL連線的話，往往你會需要下載MySQL在你自己的OS系統
，假如你的程式要給別人用或進行測試的話，他們也會需要去下載MySQL，可想而知這不是一個輕鬆的事情，而且往往會有 在自己的電腦程式跑的超好，表現超棒，換到別人的電腦後直接原地死亡 的尷尬情況

而 Docker 可以避免掉繁瑣的安裝流程，讓應用程式都運行在容器 (Container) 上，就算現在換到別人的電腦上也不會出現問題

安裝Docker

官方安裝網址 https://www.docker.com/products/docker-desktop

Hello Docker

安裝完成之後在自己的Terminal打上docker應該會就跳出

當成功出現這個畫面之後，恭喜! 你可以開始和Docker培養感情了XD

Docker image

Docker image 可以想成是一個應用，它可以是MySQL, Postgres, nginx, golang之類的

這個連結是官方用來放置Docker image的網站 https://hub.docker.com/

以MongoDB為例，在docker hub上搜尋 mongo，就可以看到mongo的image

點進去可以看到

在Terminal輸入 docker pull redis

Note: 在image name後面加上 :Tag 可以選擇版本 EX: docker pull redis:7.0-rc1 不輸入Tag代表使用最新的版本

下載完成後輸入 docker image ls 可以列出目前已經下載的image

像是

REPOSITORY TAG IMAGE ID CREATED SIZE
redis latest f1b6973564e9 3 weeks ago 113MB

當你下載了之後不喜歡，或是覺得用不到了可以使用 docker rmi [image name/ image ID] 去移除掉

使用Docker後的注意事項

Posted on 2022-02-22 In Postgres , Docker

前陣子使用到Docker上的PostgreSQL來當作我目前開發的應用的Database

在自己的筆電上都運行的好好的沒什麼事，結果隔天到實驗室之後想說用實驗室的電腦測試卻出現了Error

心想: 哭啊 出事了阿伯

時間又過了兩天，我在我的工作管理員發現了本地端的Postgres一直偷偷的在背後執行

再看看自己的docker compose file

發現問題出在ports的部分，因為postgres 初始的 port 是在5432，如果在docker上也是listen在同一個port就會發生衝突

解決這個問題的辦法有兩個，一個是把本地端的Postgres刪掉，一個是把compose file的port改掉

不過我自己因為太氣所以就直接把本地端的Postgres給砍了就是了XD

什麼是 Kubernetes?
